Instructions

You have been hired by a startup, BIN/CA$H, to do a security assessment of their new webapp. They have asked you to analyze the site for any security vulnerabilities which would let users take malicious actions.

You have not been given permission to attack or deface the site outside of the attacks given to you. You should avoid any malicious actions that are not part of the exercise.

You are given a list of potential threats, which are tracked on the left side of the screen. Your goal is to demonstrate how an attacker could perform each attack, so the company knows how to fix them.

SQL Injection

View the "private" profiles which are not normally visible

SQL UNION Attack

Find out what Alice's balance is

This attack is a slightly more advanced version of the attack in Lab 3. You may want to refer back to this guide from that lab.
Each user's balance is stored in a column named "balance". The goal will be marked as complete once Alice's balance appears anywhere on the page. Usernames are case-sensitive.

Client-side security

Send a transaction which steals money from another user

CSRF

Create a local HTML file which exploits CSRF to send you a transaction

If your attack is successful, this goal will be marked as "complete" when you open the file. It will work even though you are not allowed to send yourself a transaction.

Stored XSS

Trick the support staff into sending you money

The support staff will click any link you send them. You may need to refresh your page for the goal scoreboard to update.
